**index.css — WCAG contrast fixes:**

1. **Dark mode --text-muted (per D-09):** In the `:root.dark-mode` block, change:
```css
--text-muted: rgba(255, 255, 255, 0.4);
```
to:
```css
--text-muted: rgba(255, 255, 255, 0.50);
```
Fixes ratio from 3.77:1 to 5.37:1 (AA requires 4.5:1).

2. **Light mode --text-muted (per D-09):** In the `:root.light-mode` block, change:
```css
--text-muted: rgba(51, 47, 46, 0.45);
```
to:
```css
--text-muted: rgba(51, 47, 46, 0.70);
```
Fixes ratio from 2.30:1 to 4.75:1.

3. **Light mode --text-secondary (PLSH-02):** In the `:root.light-mode` block, change:
```css
--text-secondary: #8C867E;
```
to:
```css
--text-secondary: #6E6860;
```
Fixes ratio from 3.14:1 to 4.80:1.

4. **Light mode --banner-text (per D-11 / PLSH-01):** In the `:root.light-mode` block, change:
```css
--banner-text: #e65100;
```
to:
```css
--banner-text: #bf360c;
```
Fixes ratio from 3.13:1 to 4.62:1. Dark mode `--banner-text: #ffb74d` stays unchanged (already 10.06:1).

**App.css — Firefox scrollbar fallback (PLSH-03):**

Add immediately BEFORE the existing `::-webkit-scrollbar` rule block (around line 304):
```css
/* Firefox scrollbar */
* {
  scrollbar-color: var(--scrollbar-thumb) transparent;
  scrollbar-width: thin;
}
```

This uses the same `--scrollbar-thumb` variable that the webkit rules use, ensuring Firefox users get a themed scrollbar. The `scrollbar-width: thin` matches the 8px webkit width.
