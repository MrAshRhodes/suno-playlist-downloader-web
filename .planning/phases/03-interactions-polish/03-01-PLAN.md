**index.css changes:**

Add `--progress-glow` to the `:root.dark-mode` block (after the `--progress-track` line):
```css
--progress-glow: 0 0 8px rgba(59, 74, 107, 0.5);
```

Add `--progress-glow` to the `:root.light-mode` block (after the `--progress-track` line):
```css
--progress-glow: 0 0 8px rgba(59, 74, 107, 0.35);
```

Per D-06: This variable is already referenced by `.progress-fill { box-shadow: var(--progress-glow); }` in App.css but was never defined, so the glow was silently not rendering.

**App.css changes:**

1. **Button hover glow (per D-01):** Replace the `.btn-accent:hover:not(:disabled)` rule (around line 57-61) with:
```css
.btn-accent:hover:not(:disabled) {
  background: var(--accent-hover);
  box-shadow: var(--shadow-button), 0 0 12px rgba(59, 74, 107, 0.3);
  transform: translateY(-1px);
}
```

2. **Button active dim (per D-02):** Replace the `.btn-accent:active:not(:disabled)` rule (around line 63-66) with:
```css
.btn-accent:active:not(:disabled) {
  background: var(--accent-active);
  box-shadow: var(--shadow-button), 0 0 12px rgba(59, 74, 107, 0.15);
  transform: translateY(0);
}
```

3. **Progress track height (per D-05):** Replace the `.progress-track` rule (around line 174-179) with:
```css
.progress-track {
  height: 6px;
  background: var(--progress-track);
  border-radius: 6px;
  overflow: hidden;
}
```

4. **Progress fill border-radius (per D-05):** In the `.progress-fill` rule (around line 181-187), change `border-radius: 4px;` to `border-radius: 6px;`. The `box-shadow: var(--progress-glow);` line stays as-is (it already references the variable, which is now defined).

5. **Mantine Button hover glow override:** Replace the existing `.mantine-Button-root[data-variant="filled"]:hover` rule (around line 341-343) with:
```css
.mantine-Button-root[data-variant="filled"]:hover {
  background-color: var(--accent-hover) !important;
  box-shadow: var(--shadow-button), 0 0 12px rgba(59, 74, 107, 0.3) !important;
}
```

6. **Mantine Button active glow override:** Replace the existing `.mantine-Button-root[data-variant="filled"]:active` rule (around line 344-346) with:
```css
.mantine-Button-root[data-variant="filled"]:active {
  background-color: var(--accent-active) !important;
  box-shadow: var(--shadow-button), 0 0 12px rgba(59, 74, 107, 0.15) !important;
}
```

7. **Mantine Loader accent override (Pitfall 3):** Add a new rule after the Mantine overrides section. Mantine v6 Oval loader renders `<svg stroke={color}>` as an inline SVG attribute -- the CSS `color` property does NOT cascade to SVG `stroke`. Target `stroke` directly:
```css
.mantine-Loader-root {
  stroke: var(--accent) !important;
}
```
This overrides the inline `stroke` attribute on the Loader SVG to use the accent color instead of Mantine's default blue. The Loader component in StatusIcon.tsx has `color="blue"` hardcoded; the CSS `!important` override beats the inline attribute and keeps changes CSS-only.
