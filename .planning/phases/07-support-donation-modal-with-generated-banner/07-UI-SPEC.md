## Interaction Contract

### Modal Trigger Logic

**localStorage key:** `suno-download-count` (verified not in use — RESEARCH.md Pitfall 3)

**Trigger formula:** `count === 1 || count % 5 === 0`

**Verified sequence:** 1, 5, 10, 15, 20...

**Trigger points:** Called at the very start of `downloadPlaylist()` in App.tsx, before `setIsDownloading(true)` and before any `await`. Modal is non-blocking — download runs concurrently.

**State owned by:** App.tsx — `const [donationModalOpen, setDonationModalOpen] = useState(false)`

### Dismiss Behavior (D-06)

- X button (withCloseButton): closes modal, does NOT increment or reset counter
- Click outside (closeOnClickOutside): same behavior as X
- No "don't show again" checkbox — modal re-appears per trigger formula
- No permanent opt-out mechanism

### Hover States

| Element | Hover Behavior |
|---------|---------------|
| CTA Button | `var(--accent-hover)` bg + `var(--shadow-button)` + `0 0 12px rgba(59,74,107,0.3)` glow — inherits from `.mantine-Button-root[data-variant="filled"]:hover` in App.css |
| X close button | Mantine default ActionIcon hover — inherits from `.mantine-ActionIcon-root` override in App.css |
| Modal overlay | Mantine default scrim — `rgba(0,0,0,0.5)` backdrop |

### Animation

| Transition | Value |
|------------|-------|
| Modal open/close | Mantine v6 default — fade + scale (no override needed) |
| Button hover color | 0.2s ease (matches `--transition-speed: 0.2s` in index.css) |
