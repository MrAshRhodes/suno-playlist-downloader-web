# Phase 06.1: Match Site Styling to Banner - Context

**Gathered:** 2026-04-14
**Status:** Ready for planning

<domain>
## Phase Boundary

Harmonize the color palette, card surfaces, buttons, borders, and overall aesthetic across both themes so the rest of the app feels like it belongs with the dark navy/purple/cyan neon hero banner. CSS variable updates and card border treatment only — no layout changes, no new components, no functional changes.

</domain>

<decisions>
## Implementation Decisions

### Palette Direction
- **D-01:** Dark mode background shifts from near-black (#0A0A0A) to deep navy (#0B0D1A). Cards shift from #1A1A1A to navy-tinted #141828. Card hover becomes #1C2035. Banner sits naturally on this base.
- **D-02:** Dark mode secondary bg shifts from #111111 to #0E1020 (between bg and card).
- **D-03:** Text stays #FFFFFF primary, muted text stays ~50% white opacity. No text color changes needed.

### Accent Color Strategy
- **D-04:** Dual accent system. Primary accent becomes neon purple #7B5EA7 (buttons, focus rings, primary actions). Hover: #8E71BA, active: #6A4F91, glow: rgba(123,94,167,0.25).
- **D-05:** Secondary accent is electric cyan #00C8FF (progress bars, highlights, secondary glows). Secondary glow: rgba(0,200,255,0.2).
- **D-06:** Contextual glow usage — buttons get purple glow on hover, progress bar gets cyan glow. Interactive elements primarily purple, status/feedback elements primarily cyan.

### Card & Border Treatment
- **D-07:** Cards keep solid navy surfaces (Monolith solid-surface principle intact). No gradients on card backgrounds.
- **D-08:** Card border shifts from neutral rgba(255,255,255,0.08) to faint purple glow rgba(123,94,167,0.15) in dark mode. Subtle neon accent without breaking flat design.
- **D-09:** Shadow and inset glow stay structurally the same. Card radius stays 24px.

### Light Mode
- **D-10:** Light mode shifts from Warm Ivory to cool lavender-gray. Background: #F0F2F8, cards: #E4E7F0, card hover: #D8DCE8. Cohesive with neon dark mode — same app in daylight.
- **D-11:** Light mode text shifts to dark navy #1A1A2E (from warm brown #332F2E). Muted text: rgba(26,26,46,0.60).
- **D-12:** Light mode accent deepens for contrast — primary purple: #6B4F96, secondary cyan: #0099CC. Same hue family, darker for readability on light backgrounds.
- **D-13:** Light mode card border: rgba(107,79,150,0.12) — faint purple tint matching dark mode treatment.

### Claude's Discretion
- Exact shadow values and glow intensities can be tuned during implementation as long as they match the decided direction (navy base, purple/cyan dual accent, subtle neon borders)
- Button shadow color updates to match new accent (purple-tinted shadow instead of steel-blue)
- Scrollbar thumb colors — shift to match new palette
- Input field background and focus ring — update to purple accent
- Banner (support) colors — keep orange CTA distinction but verify contrast against new backgrounds
- Progress track color — update to match new palette

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `client/src/index.css` CSS variable system — All theming driven by `:root.dark-mode` and `:root.light-mode` variable blocks. Palette change is primarily a variable value swap.
- `client/src/App.css` `.monolith-card` class — Already consumes `var(--bg-card)`, `var(--border-color)`, `var(--shadow-card)`. Border color update flows through existing variable.
- `client/src/App.css` `.btn-accent` — Consumes `var(--accent)`, `var(--accent-hover)`, `var(--accent-active)`, `var(--shadow-button)`. Accent swap flows through.

### Established Patterns
- **CSS variables drive all theming** — classList on `<html>` swaps variable sets. This phase is primarily updating variable VALUES, not adding new mechanisms.
- **Mantine v6 overrides via `!important`** — Proven pattern for overriding framework components. Must continue for any Mantine elements that render accent colors.
- **0.3s ease transitions** — Consistent timing. No changes to transition behavior.
- **No CSS modules** — Plain CSS classes in App.css/index.css. No build tooling changes.

### Integration Points
- `client/src/index.css` lines 27-51 — Dark mode variable block. All dark palette values update here.
- `client/src/index.css` lines 54-78 — Light mode variable block. All light palette values update here.
- `client/src/App.css` — Hardcoded rgba values in hover/glow effects (e.g., `rgba(59,74,107,...)`) must be updated to new purple/cyan values or moved to CSS variables.
- `client/src/main.tsx` — AppWrapper may have hardcoded fallback colors that need updating.
- New CSS variables needed: `--accent-secondary`, `--accent-secondary-glow` (for dual accent system).

</code_context>
