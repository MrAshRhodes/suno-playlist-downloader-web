### Accent Reserved-For (both modes)

**Primary accent (purple):** `.btn-accent` background, `.btn-accent:hover` glow, `.mantine-Button-root` overrides, `.input-field:focus` border and focus ring, `.step-number` background circles, primary interactive controls.

**Secondary accent (cyan):** `.progress-fill` background, `.mantine-Progress-bar` override, `.mantine-Loader-root` stroke, progress glow effects. Cyan is reserved for progress/status feedback only — never used on buttons or action controls.

**Orange banner accent:** `.support-banner` text/border only. Intentionally distinct CTA, never used elsewhere.

**Destructive:** Not applicable in this phase. No destructive actions are introduced.

## Interaction Contracts

All interaction timing is unchanged from Phase 3. The palette swap applies to these existing interaction states:

| Interaction | Element | State | Visual Contract |
|------------|---------|-------|----------------|
| Button hover | `.btn-accent` | `:hover` | Background `var(--accent-hover)`, glow `0 0 12px rgba(123,94,167,0.30)`, translateY(-1px) |
| Button active | `.btn-accent` | `:active` | Background `var(--accent-active)`, glow `0 0 12px rgba(123,94,167,0.15)`, translateY(0) |
| Input focus | `.input-field` | `:focus` | Border `var(--accent)`, shadow `0 0 0 3px var(--accent-glow)` |
| Card hover | `.monolith-card` | hover (JS) | Background `var(--bg-card-hover)`, 0.3s ease |
| Progress fill | `.progress-fill` | active download | Background `var(--accent-secondary)`, glow `var(--progress-glow)` (cyan) |
| Theme toggle | `.theme-toggle` | `:hover` | Background `var(--bg-card-hover)`, text `var(--text-primary)`, 0.2s ease |

Transition timing: 0.2s ease for button/input interactions, 0.3s ease for background/border/shadow theme transitions. Unchanged from Phase 3.
