
   ```
   npm install --save-dev vite@latest @vitejs/plugin-react@latest
   ```

3. Check if rollup needs explicit bumping (vite 6 bundles rollup 4.x internally):

   ```
   npm audit fix
   ```
   Then check remaining issues:
   ```
   npm audit
   ```
   If HIGH vulns remain, use `npm audit fix --force` cautiously.

5. Check vite.config.ts for breaking changes. Key vite 4->6 migration points:
   - `server.allowedHosts: true` -- In vite 6, the `allowedHosts` option may have changed. If the build or dev server warns about it, change to `server.allowedHosts: 'all'` or remove if not needed for production build.

   - If vite 6 introduces any new required fields or deprecates used options, fix them.

6. Test the client build from the client directory:
   ```
   npm run build
   ```
   This must succeed. If it fails:
   - Check for breaking import changes in vite 6
   - Check for postcss compatibility (postcss 8.x should work fine with vite 6)
   - Check postcss-preset-mantine compatibility -- if it fails, bump postcss-preset-mantine too
   - Fix any TypeScript errors from updated vite types

7. Return to project root and run the full build from root:
   ```
   cd ..
   npm run build
   ```
   This runs `cd client && npm install && npm run build` per the root package.json scripts.
