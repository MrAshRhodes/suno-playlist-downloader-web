## Color

**Phase 1 scope:** These are the exact values extracted from App.tsx inline ternaries and body style assignments. They become CSS custom properties in index.css. No new palette values are introduced.

### Dark Mode (`:root.dark-mode`)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a1a` | Page background, body background |
| Secondary (30%) | `#2c2c2e` | Card backgrounds, info box, song table container |
| Input fill | `#3a3a3c` | Text input background |
| Accent (10%) | `#1a82e2` | CTA buttons, track number badge, info icon |
| Text primary | `#f5f5f7` | Body text, table row text |
| Text secondary | `rgba(255,255,255,0.7)` | Column header text |
| Text tertiary | `rgba(255,255,255,0.5)` | Tag/metadata text, placeholder |
| Border | `rgba(255,255,255,0.1)` | Card borders, table row dividers |
| Progress track | `rgba(255,255,255,0.1)` | Progress bar background |
| Button shadow | `rgba(0,0,0,0.2)` | CTA button drop shadow |

### Light Mode (`:root.light-mode`)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f5f5f7` | Page background, body background |
| Secondary (30%) | `#ffffff` | Card backgrounds, info box, song table container |
| Input fill | `#ffffff` | Text input background |
| Accent (10%) | `#0071e3` | CTA buttons, track number badge, info icon |
| Text primary | `#1d1d1f` | Body text |
| Text secondary | `#666` | Column header text, tag/metadata text |
| Border | `#e0e0e0` | Card borders |
| Table row border | `#eee` | Table row dividers |

### CSS Custom Properties Being Introduced

The following properties will be added to `:root.dark-mode` and `:root.light-mode` blocks in `index.css`, replacing all inline `theme === 'dark' ? ... : ...` occurrences in App.tsx:

```css
/* Dark mode additions */
:root.dark-mode {
  --accent-color: #1a82e2;
  --input-bg: #3a3a3c;
  --text-primary: #f5f5f7;
  --text-secondary-muted: rgba(255, 255, 255, 0.7);
  --text-tertiary: rgba(255, 255, 255, 0.5);
  --text-on-accent: #ffffff;
  --button-shadow: 0 2px 5px rgba(0, 0, 0, 0.2);
  --badge-shadow: 0 1px 3px rgba(0, 0, 0, 0.3);
  --progress-fill: #1a82e2;
  --body-bg: #1a1a1a;
  --body-text: #f5f5f7;
}

/* Light mode additions */
:root.light-mode {
  --accent-color: #0071e3;
  --input-bg: #ffffff;
  --text-primary: #1d1d1f;
  --text-secondary-muted: #666;
  --text-tertiary: rgba(0, 0, 0, 0.3);
  --text-on-accent: #ffffff;
  --button-shadow: none;
  --badge-shadow: none;
  --progress-fill: #0071e3;
  --body-bg: #f5f5f7;
  --body-text: #1d1d1f;
}
```

**Accent reserved for:** CTA buttons ("Get playlist songs", "Download as ZIP"), track number badge pill, info icon color. Nothing else.

**Destructive:** No destructive actions in Phase 1.

## Interaction Contract

**Phase 1 introduces zero new interactions.** The following existing interactions must continue to work identically after the refactor:

| Interaction | Expected Behavior | Validation |
|-------------|-------------------|------------|
| Theme toggle click | Switches class on `<html>` via `classList` (not `.className =`) | THME-05 fix |
| Theme class applied | `document.documentElement.classList.remove('dark-mode', 'light-mode'); classList.add(...)` | Replaces clobber bug |
| Body style assignment | Remove `document.body.style.backgroundColor` and `document.body.style.color` inline assignments — use CSS variables instead | Cleans up App.tsx useEffect |
| All existing UI elements | Visually identical to pre-refactor in both themes | No regressions |

```js
document.documentElement.classList.add(theme === 'dark' ? 'dark-mode' : 'light-mode');
```
