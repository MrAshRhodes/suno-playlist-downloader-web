
Place at `public/sitemap.xml`. Survives deploy.sh for same reason as robots.txt.

**C) Wire OG image into Vite build pipeline (per D-02, D-11):**
The orchestrator has already generated `public/assets/og-card.png` (1200x630px). However, `deploy.sh` runs `rm -rf public/assets` before copying, which would delete it on next deploy.

To make it permanent:
1. Create directory `client/public/assets/` (mkdir -p)
2. Copy `public/assets/og-card.png` to `client/public/assets/og-card.png`

Vite treats `client/public/` as the static public directory. Files in `client/public/` are copied as-is (no hash) to `client/dist/` during `npm run build`. So `client/public/assets/og-card.png` becomes `client/dist/assets/og-card.png`, which deploy.sh copies to `public/assets/og-card.png`.

Then run the build and deploy pipeline:
```bash
cd client && npm run build && cd ..
rm -rf public/assets
cp -r client/dist/* public/
```

This ensures og-card.png is in the build output and survives all future deploys.

Verify `public/assets/og-card.png` exists after the build.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
